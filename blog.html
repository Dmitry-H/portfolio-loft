  • Лучшие фреймворки CSS в 2017 году

    20 июля 2017

    На сегодняшний день CSS Bootstrap и Angular.js получили огромную аудиторию за счет тенденции фреймворков. Скорее всего это сохраниться ещё долго и фреймворки не будут отставать от "моды"

    В этой статье мы хотели бы продемонстрировать некоторые из лучших CSS-фреймворков на 2017 год , эти рамки используются несколькими разработчиками и имеют множество функций.

    Spectre

    Spectre.css - это легкая, отзывчивая и современная CSS-структура для более быстрого и расширяемого развития. Spectre предоставляет собой базовые стили для типографики, гибкую систему макета, основанную на гибких словах, компоненты и утилиты CSS с наилучшей практикой и последовательным языком разработки. Есть четыре способа начать работу с инфраструктурой Spectre CSS в ваших проектах. Вы можете вручную установить или использовать NPM, Yarn и Bower

    Mini.css

    Mini.css стремится обеспечить как можно больше скорости. Его размер меньше чем 7 КБ. Эта очень небольшая занимаемая площадь означает, что ваши веб-сайты и веб-приложения будут загружаться быстрее, но при этом отлично смотрятся на использовании современного компонента, который он предоставляет. Mini.css предоставляет вам настраиваемую систему аромата, позволяющую вам стать дизайнером без всякой тяжелой работы. Таким образом, вы можете создать свой собственный дизайн и персонализировать свои сайты любым способом!

    Waffle Grid

    Это CSS-сетка, построенная с помощью Flexbox. Сетка Waffle CSS - суперлегкая, весом всего 7 килобайт, с использованием настроек по умолчанию. С классами для средних и малых устройств сетка вафель может сделать ваши макеты полностью отзывчивыми.

  • Анимация набора текста на jQuery

    26 июля 2017

    Сегодня мы бы хотели вам рассказать о библиотеке TypeIt — бесплатном jQuery плагине. С её помощью можно имитировать набор текста. Если всё настроить правильно, то можно добиться очень реалистичного эффекта.

    Вся документация предельно проста и доступна онлайн.

    Для начала вам нужно подключить JavaScript файл TypeIt через CDN, а затем вызвать плагин, написав пару строк на jQuery.

    1. $('.type-it').typeIt({
    2.     strings: ['This is my string!']
    3. });

    Через функцию typeIt() вы можете замедлить или ускорить скорость набора текста.

    Все доступные настройки можно “пощупать” на странице плагина.

  • 3 самых известных редакторов кода HTML для новичка

    17 июля 2017

    Html - это язык гипертекстовой разметки, который является основой не только любого WEB-сайта, но и фундаментом интернета. Ведь интернет состоит из общедоступных сайтов, на которые люди заходят и получают себе нужную информацию.

    Чтобы создавать сайты или же изучать WEB языки программирования, нужно желательно решиться каким редактором вы будете пользоваться.

    Есть огромный выбор среди этих программ, но сегодня мы представим 3 самых лучших именно для новичков, которые станут будущими вебмастерами. Поверьте, у вас всё получиться!

    NotePad++

    Самый удобный и бесплатный редактор для новичков. Если вы профессиональный вебмастер, то всё равно желательно его стоит установить к себе на компьютер. Он захватил всех своим простым и понятным интерфейсом, но независимо от этого его функциональность довольно большая.

    Его главной минус то, что он подойдет только для Windows. К сожалению на MacOS и Linux его установить вы не сможете. Кстати, скачать его вы можете на официальном сайте.

    Sublime Text

    Очень приятный и удобный редактор, который подойдет не только для Windows, но и для MacOS и Linux. Он платный, но имеет бесконечную бесплатную версию. Просто если у вас бесплатная версия, то будет выскакивать окно и подсказка и будет просто мешать.

    На данный момент ею пользуются огромное количество вебмастеров и это говорит, что он один из самых известных.

    Adobe Dreamweaver

    Платный, но очень функциональный визуальный редактор от компании Adobe. Его шикарность в том, что он экономит очень много времени. С помощью его необязательно искать и указывать путь к изображению, система сама даст возможность выбрать и потом она сама пропишет. Ну, ради интереса можете протестировать. Купить можно на этой странице.

  • Почему WordPress лучше чем Joomla?

    26 августа 2017

    Приветствую Вас, уважаемые друзья. После долгой перемены решил сравнить WordPress c Joomla, т.к. они самые известные среди других движков.

    Этот урок скорее всего будет психологическим, т.к. многие люди работают с WordPress и одновременно с Joomla, но не могут решится каким CMS пользоваться.

    С названия урока можно догадатся, что автору статью (мне - Асадбеку) больше нравится WordPress. Это правда, но я выбрал WordPress исходя из всех параметров. И могу без проблем сказать себе, что я не ошибся при выборе.

    Поклонники Joomla, пожалуйста, не обижайтесь, я буду говорить только "факты".

    Сравнение WordPress с Joomla

    1) Каждый восьмой сайт сделан на Joomla, а каждый четвертый на WordPress - радует то, что количество пользователей WordPress неуклонно растёт, а вот количество пользователей Joomla тоже растёт, но не как WordPress. Многие мыслят, что количество пользователей не играет роль. Но они ошибаются, т.к. чем больше пользователей системы, тем больше разработчиков, новых плагинов, шаблонов и инновационных идей. Многие WordPress превратили в бизнес, у меня один знакомый знает только HTML, CSS, PHP и работает только WordPress, разрабатывая темы и продавая их.

    2) Поисковики любят больше WordPress, чем Joomla - почти каждый второй или третий раз после входа на сайт через поисковую систему, я вижу WordPress сайт. А вот за последнее время, я перестал видеть Joomla сайты, хотя количество вроде тоже растёт.

    3) WordPress намного быстрее чем Joomla. У меня был заказ, который сделал в предыдущей неделе. Заказчик попросил сделать только на Joomla, а после решил сравнить скорость сайта. У меня есть один узбекский сайт, на котором установлено несколько десяток плагинов и сайт который, я сделал в предыдущей неделе грузится медленнее чем мой сайт. Ещё радует то, что количество плагинов на моём сайте намного больше...

    Думаю, этого для начала достаточно. Но у Jooma есть один плюс - функциональность. Но позвольте спросить, зачем вы пользуетесь CMS ? Наверное из-за того, что вы не владеете языками как HTML, CSS и PHP или вы просто хотите построить сайт быстрее. Тогда зачем Вам функциональность ? Если хотите функциональный сайт, то CMS не для Вас. Изучайте PHP и MySQL после создавайте сами скрипты. Это не легко, но главное правильно!

    На этом я заканчиваю, думаю урок был полезен для вас. Ещё раз прошу прошения от поклонников Joomla.

  • Временная шкала на jQuery

    5 июня 2017

    jQuery плагин для создания временной шкалы.

    1.1. Установка через пакетного менеджера

    Timeline.js можно установить через Bower и NPM.

    bower install timelinejs-slider

    npm install timelinejs-slider

    1.2. Основы работы

    Подключаем jQuery:

    1. <script src="js/jquery.min.js"></script> <!-- >=1.11.2 -->
    2. <script src="js/timeline.min.js"></script>

    Подключаем css файл:

    1. <link rel="stylesheet" href="css/timeline.css">

    Html:

    1. <div class="timeline-container timeline-theme-1">
    2.   <div class="timeline js-timeline">
    3.     <div data-time="2017">your content or markup</div>
    4.     <div data-time="2016">your content or markup</div>
    5.     <div data-time="2015">your content or markup</div>
    6.     <div data-time="2014">your content or markup</div>
    7.     <div data-time="2013">your content or markup</div>
    8.   </div>
    9. </div><!-- /.timeline-container -->

    Запуск плагина:

    1. $(function(){
    2.   $('.js-timeline').Timeline();
    3. });

    Параметры

    1. $('.timeline').Timeline({
    2.   autoplay: false,
    3.   // value: boolean | Автопроигрывание
    4.   autoplaySpeed: 3000,
    5.   // value: integer | Скорость
    6.   mode: 'horizontal',
    7.   // value: horizontal | vertical, по умолчанию horizontal
    8.   itemClass: 'timeline-item',
    9.   // value: класс
    10.   dotsClass: 'timeline-dots',
    11.   // value: класс точки
    12.   activeClass: 'slide-active',
    13.   // value: класс для точки oitem
    14.   prevClass: 'slide-prev',
    15.   // value: класс предыдущей точки
    16.   nextClass: 'slide-next',
    17.   // value: класс следующей точки
    18.   startItem: 'first', // first|last|number
    19.   // value: first | last | number , по умолчанию first
    20.   dotsPosition: 'bottom',
    21.   // value: bottom | top, default to bottom
    22.   pauseOnHover: true,
    23.   // value: boolean | Пауза при наведении мыши
    24.   pauseOnDotsHover: false,
    25.   // value: boolean | Пауза при наведении мыши на точку
    26. });
